# Dedicated Item Textures

## Goal

Replace procedural or shared fallback artwork for every Atlas non-block item with
a dedicated, deterministic 16x16 PNG while preserving the existing pixel-art
style established by the PR 19 magnet blocks and armor sprites.

This is a rendering-only change. It must not alter item IDs, gameplay behavior,
recipes, durability, equipment stats, inventory ordering, saves, or controls.

## Scope

The source of truth for this pass is every `BLOCKS` entry with
`isItem: true`. The current branch contains 66 such entries.

The pass includes:

- Bed item.
- Wood, stone, iron, copper, gold, and diamond tools.
- Stick, coal, charcoal, ingots, raw ores, gems, and lapis.
- Apple.
- Oak, spruce, birch, and cherry saplings.
- Wheat seeds.
- Iron armor and Polarity Boots.
- Gold, diamond, and copper armor.

The existing five dedicated equipment PNGs remain in scope and must continue to
be generated by the same shared system:

- Iron Helmet
- Iron Chestplate
- Iron Leggings
- Iron Boots
- Polarity Boots

Placed blocks, block-face textures, slabs, stairs, plants rendered as blocks,
and the positive/negative magnet block textures are outside this item-only pass.

## Generation Approach

Extend the existing deterministic texture system:

- `src/systems/textures/pr19TexturePixels.ts` stores the item asset list,
  palettes, reusable sprite templates, and exact pixel definitions.
- `scripts/generate_pr19_textures.mjs` writes transparent 16x16 PNG files.
- The runtime procedural atlas uses the same definitions when an external PNG
  is unavailable.
- Tests compare committed PNG bytes with generated output.

Do not use AI image generation for final sprites. Exact pixel placement,
transparent backgrounds, shared silhouettes, and reproducible output are more
important than painterly variation.

## Visual Style

All item sprites must match Atlas's existing low-resolution style:

- Native 16x16 canvas.
- Fully transparent background.
- Hard one-pixel edges with no antialiasing.
- Three or four colors per primary material: shadow, base, highlight, and an
  optional deepest outline.
- Highlights come from the upper-left; shadows sit on lower-right edges.
- Shapes remain readable at inventory size and when rendered as dropped items.
- No gradients, blur, semitransparent edge pixels, or photographic texture.
- No direct copying of external game textures.

The magnet sprites are the quality reference: simple geometry, centered
features, strong contrast, and restrained material detail.

## Sprite Families

### Tools

Use shared silhouettes for:

- Pickaxe
- Axe
- Shovel
- Sword
- Hoe

Each material variant keeps the same silhouette and handle placement while
using a distinct palette:

- Wood: warm medium and dark browns.
- Stone: neutral gray with irregular edge accents.
- Iron: the existing Atlas iron palette.
- Copper: warm orange/copper with bright edge highlights.
- Gold: saturated yellow-gold with pale highlights and dark ochre shadows.
- Diamond: cyan-blue with bright aqua highlights and teal shadows.

Tool shapes should be more deliberate than the existing procedural one-pixel
lines while remaining compact enough to avoid touching tile edges.

### Armor

Use the existing iron armor silhouettes as the shape reference for helmet,
chestplate, leggings, and boots.

Create material variants for:

- Iron
- Gold
- Diamond
- Copper

Polarity Boots retain their red/blue split design.

Gold, diamond, and copper armor receive 12 dedicated atlas slots:

- Gold armor: slots 157-160
- Diamond armor: slots 161-164
- Copper armor: slots 165-168

These replace the current ingot/gem texture fallback assignments. Existing
block IDs and item IDs remain unchanged.

### Materials And Gems

Give each ingredient a distinct silhouette:

- Stick: diagonal two-pixel-width branch with highlight.
- Coal and charcoal: different chunk shapes and palettes.
- Ingots: beveled horizontal bars with material-specific shading.
- Raw iron, raw copper, and raw gold: irregular ore chunks with embedded color.
- Diamond and emerald: faceted gem silhouettes.
- Lapis Lazuli: an irregular blue shard or pile, distinct from diamond.

### Food, Plants, And Utility

- Apple: red fruit silhouette with a dark stem and green leaf.
- Bed item: compact isometric/side icon with red blanket, white pillow, and
  dark wood legs.
- Saplings: species-specific trunks and canopies; spruce is tiered, birch is
  pale-trunked, cherry includes pink blossoms, and oak uses a rounded canopy.
- Wheat seeds: a small clustered seed shape with multiple green-brown tones.

## Asset Mapping

Every `isItem: true` entry must have:

1. A texture slot in `src/data/blocks.ts`.
2. A unique item path entry in
   `src/systems/textures/textureMapping.ts`.
3. A generated PNG under `public/assets/textures/items/`.
4. A deterministic pixel definition used by both runtime fallback drawing and
   the PNG generator.

Items that already have unique atlas slots keep them. Only the 12 armor pieces
currently sharing ingot/gem slots receive new slots.

File names use lowercase snake case matching the item name, for example:

- `items/diamond_pickaxe.png`
- `items/gold_chestplate.png`
- `items/raw_copper.png`
- `items/cherry_sapling.png`

No two different items may resolve to the same PNG path.

## Runtime Behavior

`TextureAtlasManager` continues loading external images through the existing
mapping. If a PNG cannot be loaded, the atlas must render the same deterministic
pixel definition, not an unrelated generic placeholder.

The same slot and artwork must be visible in:

- Inventory and creative inventory.
- Hotbar.
- First-person held-item rendering.
- Dropped-item rendering.
- Any UI using the texture atlas.

## Tests

Add or extend automated tests to verify:

- All 66 `isItem: true` entries have an item PNG mapping.
- Every mapped PNG exists.
- Every item path is unique.
- All mapped item images are exactly 16x16 RGBA PNGs.
- Transparent pixels remain transparent and opaque sprite pixels use hard
  alpha values.
- Gold, diamond, and copper armor each use four new unique slots instead of
  ingot/gem slots.
- Shared tool silhouettes remain consistent across material variants.
- Material palettes remain distinct.
- Existing magnet, iron armor, Polarity Boots, and Iron Block texture tests
  continue passing.
- `scripts/generate_pr19_textures.mjs --check` confirms committed PNGs exactly
  match the source definitions.

Run:

```text
node --test <all test files>
npm run typecheck
npm run lint
npm run build
```

## Manual Validation

Load a creative world and inspect:

- Every creative-inventory item.
- Representative tools from each material.
- All armor pieces.
- Ingredients, gems, food, saplings, seeds, and the bed.
- Selected items in first person.
- Dropped items in daylight and darkness.

Verify that no item shows a block texture, ingot fallback, missing image, blurry
scaling, or transparent fringe.

## Compatibility

- No `BlockType` values change.
- Existing texture slots remain stable except the 12 armor entries that move
  from shared fallback slots to new slots 157-168.
- Save data remains compatible because saves store item IDs, not texture slots.
- No gameplay or physics behavior changes.
